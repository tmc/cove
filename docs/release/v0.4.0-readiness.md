# cove v0.4.0 readiness audit

Use this checklist before cutting the v0.4.0 tag. It is a pre-tag punch list,
not a release approval.

## Required checks

- [ ] `git status --short --branch` is clean on the release branch.
- [ ] `git log --oneline origin/main..HEAD` shows only release-prep commits.
- [ ] `RELEASE-NOTES-v0.4.0.md` is current and readable.
- [ ] `docs/integrations/matrix.md` is current and readable.
- [ ] Cross-reference sections were added to the shipped design docs that need
      them.
- [ ] `go build -o cove . && codesign -s - -f --entitlements internal/autosign/vz.entitlements ./cove`
      passes.
- [ ] `go test ./...` passes.
- [ ] The release checklist in `docs/reference/release-checklist.md` still
      matches the current tag flow.
