# cove v0.1.0 — publish checklist

Runbook for tagging and publishing v0.1.0. Walk top-to-bottom.

State at start:
- `main @ 706cf0d` (current local tip, **26 commits ahead of origin/main**)
- `next` branch exists locally, **not yet pushed**
- `RELEASE-NOTES-v0.1.0.md` at repo root
- Reduced 0.1 smoke: 12/12 PASS on `hermes-mlx-go-60g-v10`
- Tag does not exist locally or on origin

## 1. Pre-tag verification

```bash
git status                                    # expect: clean
git log --oneline -1                          # expect: 706cf0d (or later)
go test ./...                                 # expect: green
go build -o cove . && codesign -s - -f --entitlements internal/autosign/vz.entitlements ./cove
./cove -version                               # sanity check binary runs
```

Optional smoke replay against hermes (12 items — see `docs/ship-audit-2026-04-24.md` "Reduced smoke test" section).

## 1a. Push main to origin BEFORE tagging

`origin/main` is currently behind local `main` (26 commits as of 706cf0d).
Tag at step 2 will point at local HEAD; if origin doesn't have those
commits when the tag pushes in step 3, the tag references commits the
remote can't resolve. Push main first:

```bash
git fetch origin main
git log --oneline origin/main..HEAD | head -30  # review what you're about to publish
git push origin main                            # publish the commits the tag will reference
git ls-remote origin refs/heads/main            # expect: matches local HEAD
```

If the `git log` review shows commits you don't intend to ship in 0.1,
stop and resolve before tagging.

## 2. Tag creation

Your git config: SSH signing key set (`user.signingkey=/Users/tmc/.ssh/id_ed25519.pub`),
`commit.gpgsign=true`, `tag.gpgSign` **not** set. Use `-s` explicitly to
sign the tag with your SSH key:

```bash
git tag -s v0.1.0 -m "cove v0.1.0 — first tagged release

First tagged release. Operate-a-pre-existing-VM surface over CLI, HTTP,
and MCP. See RELEASE-NOTES-v0.1.0.md."
git cat-file -p v0.1.0 | head -20             # inspect tag object + signature header
git show v0.1.0 --stat | head -20             # sanity check
```

`git tag -v` is intentionally omitted: SSH-signed tags require
`gpg.ssh.allowedSignersFile` to be configured, which is not set in this
repo. The signature is embedded in the tag object regardless; GitHub and
the Go module proxy verify it server-side. If you want local verification,
configure `allowedSignersFile` once-and-done — see
`git config --global gpg.ssh.allowedSignersFile`.

Tag points at `706cf0d` (or whatever `main` HEAD is when you run this).

## 3. Push

`main` was already pushed in step 1a. `next` branch is not on origin yet;
tag is not on origin yet.

```bash
git push origin next                          # first push of staging branch (see roadmap branch hygiene)
git push origin v0.1.0                        # or: git push origin --tags
```

Verify:

```bash
git ls-remote origin refs/tags/v0.1.0         # expect: one line
git ls-remote origin refs/heads/next          # expect: one line
```

## 4. GitHub release

`gh` is authenticated as `tmc` with `repo` scope. One-liner:

```bash
gh release create v0.1.0 \
  --title "cove v0.1.0" \
  --notes-file RELEASE-NOTES-v0.1.0.md \
  --verify-tag
```

If you want to attach a prebuilt binary:

```bash
tar czf cove_v0.1.0_darwin_arm64.tar.gz cove internal/autosign/vz.entitlements
gh release upload v0.1.0 cove_v0.1.0_darwin_arm64.tar.gz
```

(GoReleaser usually produces these under `dist/`; if you run it, it'll
upload automatically.)

## 5. Homebrew tap update

`dist/homebrew/Casks/cove.rb` is GoReleaser-generated ("DO NOT EDIT").
Current state: `version "0.0.0-SNAPSHOT-fb9b3b4"`, sha256 from
snapshot. To update:

Option A (preferred) — let GoReleaser do it:

```bash
goreleaser release --clean                    # reads git tag, regenerates dist/
```

This regenerates `dist/homebrew/Casks/cove.rb` with the real version and
sha256, and pushes to `tmc/homebrew-tap` if the release workflow is wired.

Option B — manual, if GoReleaser isn't set up:

1. Build + tarball the release binary:
   ```bash
   tar czf cove_0.1.0_darwin_arm64.tar.gz cove
   shasum -a 256 cove_0.1.0_darwin_arm64.tar.gz
   ```
2. Edit `cove.rb` in `tmc/homebrew-tap` (not this repo):
   - `version "0.1.0"`
   - `url "https://github.com/tmc/vz-macos/releases/download/v0.1.0/cove_0.1.0_darwin_arm64.tar.gz"`
   - `sha256 "<shasum output>"`
3. Commit + push the tap repo.

Verify:

```bash
brew update && brew install tmc/tap/cove
cove -version
```

## 6. Warm the Go module proxy

Optional but cheap — saves the first `go install` user a multi-second
cold-cache wait:

```bash
GOPROXY=https://proxy.golang.org go list -m github.com/tmc/vz-macos@v0.1.0
```

Returns `github.com/tmc/vz-macos v0.1.0` once the proxy has indexed the
tag (usually within a minute of the push in step 3).

## 7. Post-publish surfaces

- **README.md** — no "pre-release" language to flip; the install
  instructions (`brew install tmc/tap/cove`) go live as soon as the tap
  update from step 5 is pushed.
- **RELEASE-NOTES-v0.1.0.md** — leave as-is; GitHub Release copies it.
- **docs/ship-audit-2026-04-24.md** — leave; historical record.
- No badges reference version; nothing else to touch.

## 8. Rollback

If the tag is bad (wrong commit, broken binary, bad notes):

```bash
# Local
git tag -d v0.1.0

# Remote tag
git push origin :refs/tags/v0.1.0

# GitHub release (deletes the release, not the tag)
gh release delete v0.1.0 --yes

# Homebrew tap: revert the cask commit in tmc/homebrew-tap
```

Then fix the underlying issue, retag, redo steps 2–6.
