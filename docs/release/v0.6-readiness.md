# cove v0.6.0 readiness audit

This audit records the release-prep gates for the v0.6.0 tag. The tag
was cut after user approval; public distribution remains user-gated.

## Headline

v0.6 raises the macOS host floor from implicit 12.0 to **14.0 (Sonoma)**
and lands design 041 ScreenCaptureKit Slices 1-3 (probe, spike, dual-path
opt-in). Slice 4 (default flip + `CGWindowListCreateImage` retire) is
deferred to v0.7 pending Slice 2 perf data. v0.6 also closes the last
purely-engineering Cirrus migration blocker by routing
`ENCRYPTED[…]` secrets through `cove shell --secret-env` and
`cove-action`'s `secrets:` input. Polish landmarks: design 040 storage
budget Phases 0-5, 53 `-h` CLI bugs squashed (R81-R84), `cove doctor`
E2E coverage (R76-R80), `cove-action` cache-mode + cache-scope, storage
census PiB int64 overflow fix.

## Release-cut command sequence

Historical command sequence used for the private tag cut.

```bash
cd /Volumes/tmc/go/src/github.com/tmc/cove
git checkout main
git pull --ff-only origin main

# Verify cut anchor.
git rev-parse HEAD

# Pre-flight: build, sign, test.
go build -o cove .
codesign -s - -f --entitlements internal/autosign/vz.entitlements ./cove
go test ./...

# Smoke: ./cove --version, ./cove doctor, ./cove image list,
#        ./cove doctor sckit-preauth (informational with default cgwindow).

# Tag (user-gated).
git tag -a v0.6.0 -m "cove v0.6.0"
# DO NOT push to origin until user approves the tag cut.
git push origin v0.6.0
```

## Pre-flight verifications

Before pushing a future patch tag:

1. Confirm the target commit is on private `origin/main`.
2. Re-run `go test ./...` including the `internal/sckit` package
   tests. Any failure halts the cut.
3. Re-sign `./cove` with `internal/autosign/vz.entitlements` after
   build. The Virtualization framework refuses to load on first run if
   the binary is not properly signed.
4. Smoke `./cove --version`, `./cove doctor`, `./cove image list`,
   `./cove doctor sckit-preauth`. None should panic; doctor exit code
   may be nonzero on hosts missing optional state (TCC, Apple Events,
   sandbox) — acceptable for the cut.
5. Confirm `RELEASE-NOTES-v0.6.0.md` and this readiness doc are
   committed on the tag commit.
6. Verify host floor messaging: a 13.x host should see the 14.0+
   advisory on first SCKit capture (paths gated; default `cgwindow`
   leaves 13.x untouched).

## Privacy gate (load-bearing)

cove repo is **private**. Until the user flips it public, do **not**:

- Push to `tmc/homebrew-tap`.
- Push to `nixpkgs`.
- Push images to public OCI registries.
- Reference public cove URLs in any Cask / formula / package metadata.

Internal feature work and private tag pushes are fine. The `git push
origin v0.6.0` step above pushes to the private cove origin only.

## Risk callouts

| Risk | Severity | Mitigation |
|---|---|---|
| 13.x host hits 14.0 floor unexpectedly | Medium | Default capture stays `cgwindow`; only `COVE_CAPTURE_BACKEND=sckit` opt-in surfaces the 14.0 advisory. Release notes call this out. |
| Slice 2 perf data still TBD | Low | Slice 4 explicitly deferred to v0.7; v0.6 ships dual-path opt-in only. |
| Secret leakage via `--secret-env` | Medium | Metrics redactor scrubs resolved values from `cove runs` logs. No in-band stdout/stderr masking; documented in RELEASE-NOTES. |
| Image gc R6 cache TTL still open | Low | Deferred; not on the gc-race hot path. |
| Future patch tag drifts from green main | Medium | Re-run build, sign, and test on the exact private tag commit before pushing. |

## Rollback posture

A v0.6.0 tag annotated against `dd5f58f`. If a post-tag
smoke uncovers a P0:

1. **Tag deletion** (cheap, no public consumers because privacy gate
   stands):
   ```bash
   git tag -d v0.6.0
   git push --delete origin v0.6.0
   ```
2. Land the fix on `main`, retag at the new HEAD.

If the privacy gate has been flipped before discovery:

1. Cut `v0.6.1` on `main` with the fix.
2. Update `RELEASE-NOTES-v0.6.0.md` to record the issue and link to
   v0.6.1.
3. Do not delete the public tag.
