## Release-cut command sequence

Run from the release-prep worktree, on the merge commit you intend to tag.

```bash
cd /Volumes/tmc/go/src/github.com/tmc/vz-macos
git checkout main
git pull --ff-only origin main

# Verify code-complete anchor.
git rev-parse HEAD   # must equal 8bd7a65 or descend from it on main

# Pre-flight: build, sign, test.
go build -o cove .
codesign -s - -f --entitlements internal/autosign/vz.entitlements ./cove
go test ./...

# Smoke: ./cove --version, ./cove doctor, ./cove image list.

# Tag (user-gated).
git tag -a v0.5.0 -m "cove v0.5.0"
# DO NOT push to origin until user approves #224 / #225 / #294.
git push origin v0.5.0
```

## Pre-flight verifications

Before pushing the tag:

1. Confirm `main` HEAD matches the intended anchor (currently `8bd7a65` or
   any descendant containing only doc/release-prep commits).
2. Re-run `go test ./...`. Two pre-existing failures should be green at
   `1f8435c`. Any other failure halts the cut.
3. Re-sign `./cove` with `internal/autosign/vz.entitlements` after the build.
   The Virtualization framework refuses to load on first run if the binary is
   not properly signed.
4. Smoke `./cove --version`, `./cove doctor`, `./cove image list`. None should
   panic; doctor exit code may be nonzero on hosts missing optional state
   (TCC, Apple Events, sandbox), which is acceptable for the cut.
5. Confirm `RELEASE-NOTES-v0.5.0.md` and `docs/release/v0.5-readiness.md` are
   committed on the tag commit.

## Privacy gate (load-bearing)

cove repo is **private**. Until the user flips it public, do **not**:

- Push to `tmc/homebrew-tap` (the cove cask there will 404 for
  non-collaborators).
- Push to `nixpkgs`.
- Push images to public OCI registries.
- Reference public cove URLs in any Cask / formula / package metadata.

Internal feature work and private tag pushes are fine. The `git push origin
v0.5.0` step above pushes to the private cove origin only.

## Rollback posture

A v0.5.0 tag is annotated and signs against `8bd7a65` (or descendant). If a
post-tag smoke uncovers a P0:

1. **Tag deletion** (cheap, no public consumers because privacy gate stands):
   ```bash
   git tag -d v0.5.0
   git push --delete origin v0.5.0
   ```
2. Land the fix on `main`, retag at the new HEAD.

If the privacy gate has been flipped before discovery:

1. Cut `v0.5.1` on `main` with the fix.
2. Update `RELEASE-NOTES-v0.5.0.md` to record the issue and link to v0.5.1.
3. Do not delete the public tag.
