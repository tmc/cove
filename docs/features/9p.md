# 9p host export

`cove 9p serve` exposes a read-only 9p view of local cove VM metadata.
It is a host management surface. It does not mount a guest filesystem and
does not replace VirtioFS shared folders.

```sh
cove 9p serve -addr unix:/tmp/cove.9p
```

The exported tree starts with:

```text
/
  vms/
    <name>/
      config.json
      state.json
      shared_folders.json
      disks.json
```

`state.json` records the VM name, registry path, detected OS, stopped or
suspended state, and disk size. `disks.json` lists known cove disk files by
name, host path, and size. The first implementation exports disk metadata
only; it deliberately does not export live disk contents.

The default address is `unix:/tmp/cove.9p`. TCP listeners must bind to
`localhost`, `127.0.0.1`, or `::1`.

Use this for tools that need to inspect cove's VM registry through a filesystem
protocol. Use `cove shared-folder` for guest-visible host paths.
